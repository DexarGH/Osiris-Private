# Osiris - CS2 Game Hack

## Что это
Osiris — это кроссплатформенный (Windows, Linux) чит для Counter-Strike 2, основанный на перехвате внутреннего рендеринга игры и манипуляции Panorama UI. Совместим с последним обновлением CS2 в Steam.

**Текущий форк:** https://github.com/DexarGH/Osiris-Private
**Оригинал:** https://github.com/danielkrupinski/Osiris (Daniel Krupiński)

## Где запускается
- **ОС:** Linux (Arch Linux), Windows
- **Игра:** Counter-Strike 2 (Steam)
- **Инжект:** `build/Source/inject.sh` (Linux) — использует GDB для инжекции `.so` в процесс `cs2`

## Технологии
- **C++20** (минимум g++ 11 или clang++ 18)
- **CMake 3.24+**
- **Без CRT** в Release (нет стандартной библиотеки C++, нет heap аллокаций, нет потоков, нет исключений)
- **SDL3** — только для перехвата событий (input hook)
- **VMT Swapping** — перехват виртуальных методов
- **Pattern Scanning** — поиск адресов в памяти по сигнатурам

## Как работает чит

1. **Инжекция** → `inject.sh` вызывает GDB, который загружает `libOsiris.so` в процесс `cs2`
2. **Инициализация** → `SDLHook_PeepEvents` перехватывает события, запускает `finishInit()`
3. **Хуки** → `ViewRenderHook` перехватывает `OnRenderStart` (VMT swap)
4. **Каждый кадр** → `ViewRenderHook_onRenderStart()` вызывает все фичи:
   - Aimbot
   - FOV Circle
   - World Particles (РАБОТАЕТ — Tail режим, см. ниже)
   - Glow
   - HUD элементы
   - GUI

## Текущие реализованные фичи (РАБОТАЮТ ✅)

### AimBot (Combat -> Aim Bot)
- **Enable** (Yes/No)
- **Bind** — выбор любой кнопки (109 вариантов)
- **Bind Mode** — Hold/Toggle/Always On
- **Smooth** (0-20) — плавность (0=быстро, 20=плавно)
- **Rotation** — Linear/Sigmoid
- **FOV** (0-180) — угол обзора
- **Multi Point Size** (0.00-1.00)
- **Visible Checks** (Yes/No) — заглушка, нужен паттерн TraceRay
- **Flash Checks** (Yes/No) — РЕАЛИЗОВАНО через `getRemainingFlashBangTime()`

### FOV Circle (HUD -> FOV Circle)
- **Enable** (Yes/No)
- **Thickness** (1-10) — толщина рамки
- **Color** — 7 цветов (Green/Red/Blue/Yellow/Cyan/Magenta/White)
- Берёт FOV автоматически из AimBot настроек
- Использует Panorama UI панель с border-radius

### HUD элементы
- Bomb Timer
- Bomb Plant Alert
- Defusing Alert
- Killfeed Preserver
- Post-round Timer

### Visuals
- Model Glow
- Outline Glow
- Player Info In World (ESP)
- Viewmodel Modification

## World Particle

### Что работает
- **Enable** (Yes/No)
- **Count** (1-500) — количество частиц
- **Mode** — Tail (работает), Rain (НЕ РЕАЛИЗОВАН — скрывает частицы)
- **Particle** — только Star (SVG: `star.svg`), остальные заготовлены но не используются
- **Color** — Hue слайдер (0-360°) с квадратиком превью цвета

### Текущая проблема
- **Цвет существующих частиц не обновляется при смене Hue**: новые частицы тоже спавнятся с старым цветом
- Нужно: при изменении Hue слайдера вызвать `updateParticleColors()` для всех активных частиц
- Метод `updateParticleColors()` уже есть в коде, но не вызывается из GUI handler'а

### Архитектура
- SVG: `s2r://panorama/images/icons/ui/star.svg`
- Позиционирование: `WorldToClipSpaceConverter` → `PanoramaTransformations` (как FootstepVisualizer)
- Цвет: `setWashColor()` на ImagePanel
- Контейнер: `hudReticle` → `fitParent()` → панели внутри
- Плавное затухание: opacity на основе времени жизни (2 секунды)
- Движение игрока: delta позиции между кадрами, порог 0.05

### Что попробовать дальше
1. Вызвать `updateParticleColors()` из `PanoramaGUI::onWorldParticleHueSliderValueChanged()`
2. Реализовать Rain режим (частицы падают сверху экрана)
3. Добавить новые типы частиц (Snow, Bloom, Dollar) когда найдём рабочие SVG

## Логирование

### Где логи
`~/OsirisCS2/logs/osiris_YYYY-MM-DD.log`

### Как работает
- Авто-создание при первом вызове `LOG_INIT()` в `finishInit()`
- Лимит 5MB на файл (потом останавливается)
- Один файл в день с датой в имени

### Использование
```cpp
#include <Utils/Logger.h>
LOG("[Feature] Message: %d, %.2f", intValue, floatValue);
LOG_INIT();  // В init
```

## Как добавить новую фичу

### 1. Создать конфигурационные переменные
`Source/Features/MyFeature/MyFeatureConfigVariables.h`:
```cpp
#pragma once
#include <Config/ConfigVariable.h>
#include <Config/RangeConstrainedVariableParams.h>

namespace my_feature_vars {
    CONFIG_VARIABLE(Enabled, bool, false);
    // ... другие переменные
}
```

### 2. Создать состояние (если нужно)
`Source/Features/MyFeature/MyFeatureState.h`:
```cpp
#pragma once
#include <CS2/Panorama/PanelHandle.h>

struct MyFeatureState {
    cs2::PanelHandle panelHandle{};
};
```

### 3. Создать основной класс
`Source/Features/MyFeature/MyFeature.h`:
```cpp
#pragma once
#include "MyFeatureConfigVariables.h"
#include "MyFeatureState.h"

template <typename HookContext>
class MyFeature {
public:
    explicit MyFeature(HookContext& hookContext) noexcept : hookContext{hookContext} {}

    void update() const {
        if (!GET_CONFIG_VAR(my_feature_vars::Enabled))
            return;
        // Логика фичи
    }

    void onUnload() const {
        // Очистка
    }

private:
    HookContext& hookContext;
};
```

### 4. Добавить в FeaturesStates
`Source/Features/FeaturesStates.h`:
```cpp
#include "MyFeature/MyFeatureState.h"
struct FeaturesStates {
    // ...
    MyFeatureState myFeatureState;
};
```

### 5. Добавить в ConfigSchema
`Source/Config/ConfigSchema.h` — в нужный объект:
```cpp
configConversion.boolean(u8"MyFeatureEnabled", loadVariable<my_feature_vars::Enabled>(), saveVariable<my_feature_vars::Enabled>());
```

### 6. Добавить в ConfigVariableTypes
`Source/Config/ConfigVariableTypes.h`:
```cpp
#include <Features/MyFeature/MyFeatureConfigVariables.h>
// В список TypeList:
my_feature_vars::Enabled,
```

### 7. Добавить include в GlobalContext
`Source/GlobalContext/GlobalContext.h`:
```cpp
#include <Features/MyFeature/MyFeature.h>
```

### 8. Добавить вызов в EntryPoints
`Source/EntryPoints/EntryPoints.h`:
```cpp
hookContext.make<MyFeature>().update();
```

### 9. Добавить GUI (если нужно)
`Source/UI/Panorama/CreateGUI.js`:
```javascript
createYesNoDropDown(tab, "Enable MyFeature", 'section', 'my_feature_enabled');
```

### 10. Добавить обработчики в Tab
`Source/UI/Panorama/VisualsTab.h` (или HudTab.h / CombatTab.h):
- Добавить `initDropDown<Handler>(guiPanel, "my_feature_enabled")`
- Добавить `setDropDownSelectedIndex(mainMenu, "my_feature_enabled", !GET_CONFIG_VAR(...))`

## Полезные макросы

- `GET_CONFIG_VAR(Variable)` — получить значение переменной конфига
- `SET_CONFIG_VAR(Variable, value)` — установить значение
- `CONFIG_VARIABLE(Name, Type, Default)` — объявить переменную
- `CONFIG_VARIABLE_RANGE(Name, Params)` — объявить переменную с диапазоном
